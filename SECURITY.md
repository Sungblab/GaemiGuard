# 보안 정책

GaemiGuard는 계좌, 주문, 인증 토큰, 투자 기록을 다룰 수 있는 프로젝트입니다. 보안 이슈는 일반 버그보다 우선순위가 높습니다.

## 지원 범위

현재는 `main` 브랜치와 활성 개발 PR만 지원합니다. 아직 정식 릴리스는 없습니다.

## 공개 이슈에 올리면 안 되는 것

- API key, OAuth client secret, access token, refresh token
- 계좌번호, 주문번호, 거래 내역 원본
- 개인 식별 정보
- 실제 인증 요청/응답 전문
- 취약점 악용에 바로 사용할 수 있는 상세 절차

## 취약점 제보

공개 이슈 대신 GitHub Security Advisory 또는 저장소 관리자에게 비공개 채널로 제보해 주세요.

제보에 포함하면 좋은 내용:

- 영향 범위
- 재현 조건
- 기대 동작과 실제 동작
- 가능한 경우 최소 재현 코드
- 민감 정보를 제거한 로그

## 기본 보안 방향

- 실주문은 기본적으로 차단합니다.
- 자동매매는 별도 권한, 감사 로그, kill switch 없이는 활성화하지 않습니다.
- 에이전트 도구 실행은 권한 단계별로 제한합니다.
- 민감 정보는 로그와 artifact에 원문으로 남기지 않습니다.
- 외부 API 연동은 공식 문서 기반으로 구현합니다.
