# GaemiGuard

GaemiGuard는 한국 개인투자자를 위한 로컬 개인 투자 에이전트입니다.

토스증권 같은 매매 화면을 베끼려는 프로젝트가 아닙니다. 블룸버그 터미널처럼 모든 시장 데이터를 한꺼번에 쏟아내는 프로젝트도 아닙니다. 핵심은 내 계좌, 관심 종목, 리서치, 시나리오, 투자 원칙을 에이전트가 한곳에 모아 판단 전에 확인할 근거를 남기는 것입니다.

한 줄로 말하면 이렇습니다.

> 판단 점검부터 주문 실행까지 GaemiGuard에서 하되, 실제 연결은 공식 증권사 API로만 합니다.

GaemiGuard 안에는 세 가지 성격이 함께 있습니다.

- 개인 투자 에이전트: Commander가 질문을 받고 필요한 에이전트와 도구를 부릅니다.
- 투자 점검 가드: 주문 전 내 원칙, 노출, 경고, 근거 부족을 확인합니다.
- 작은 로컬 투자 터미널: 에이전트가 본 계좌/시장/리서치 근거를 화면에 보여줍니다.

중요한 순서는 에이전트가 먼저이고, 터미널 화면은 그 에이전트가 사용한 근거를 보여주는 보조 화면입니다.

증권사는 갈아끼울 수 있어야 합니다. Toss, 한국투자, 키움, LS 같은 연결은 GaemiGuard의 본체가 아니라 브로커 어댑터입니다. 증권 연결이 없어도 관심종목, 투자 원칙, 수동 포트폴리오, 샘플 데이터로 먼저 써볼 수 있어야 합니다.

## 무엇을 하나요?

GaemiGuard는 투자자가 주문 버튼을 누르기 전후의 흐름을 한곳에 모읍니다.

- 브로커 계좌, 수동 포트폴리오, 관심종목을 함께 봅니다.
- 내 투자 원칙, 보유 비중, 현금, 환율, 시장 일정, 경고를 확인합니다.
- Hermes 같은 리서치 도구와 MiroFish 같은 시나리오 도구를 에이전트가 필요할 때 부릅니다.
- 리서치 결과, 시나리오, 주문 검토, 사용자의 판단 이유를 로컬에 남깁니다.
- 주문 초안과 모의 주문으로 먼저 점검합니다.
- 사용자가 승인한 수동 실주문을 공식 브로커 API로 실행합니다.
- 사용자가 정한 원칙, 한도, 중단 스위치 안에서만 자동매매를 다룹니다.

핵심은 “더 빨리 사고팔기”가 아니라 “내가 왜 이 주문을 하려는지, 위험한 부분은 없는지, 나중에 다시 확인할 근거가 남는지”입니다.

## 왜 만들고 있나요?

개인투자에서 위험한 순간은 보통 “기능이 없어서”가 아니라 “생각할 틈 없이 눌러서” 생깁니다.

예를 들면 이런 상황입니다.

- 방금 본 뉴스 하나만 보고 급하게 주문하려고 할 때
- 내 계좌가 이미 같은 업종에 많이 노출되어 있는지 모를 때
- 손실을 만회하려고 평소 원칙보다 큰 금액을 넣으려 할 때
- AI에게 물어봤지만 그 답이 어떤 자료를 근거로 했는지 남지 않을 때

GaemiGuard는 이런 흐름을 로컬 앱 안에서 붙잡는 쪽을 목표로 합니다.

- 내 계좌와 관심 종목을 한 화면에서 보고
- 리서치와 시나리오를 함께 남기고
- 에이전트가 도구를 실행하더라도 권한과 기록을 남기고
- 주문은 안전 장치, 명시 승인, 감사 로그, 중단 스위치 안에서만 실행합니다.

## 빠르게 실행하기

필요한 것:

- Node.js 22 이상
- pnpm 10 이상

설치:

```powershell
pnpm install
```

개발 서버 실행:

```powershell
pnpm dev
```

기본 주소:

- API: `http://127.0.0.1:4317`
- 데스크톱 화면 개발 서버: `http://127.0.0.1:5173`

검증:

```powershell
pnpm verify
```

문서를 바꿨다면 이것도 실행합니다.

```powershell
pnpm docs:agent-check
pnpm docs:html
```

화면 흐름을 바꿨다면 Windows에서는 아래 명령으로 확인합니다.

```powershell
pnpm smoke:desktop
```

## 에이전트에게 맡겨서 설치하기

Codex, Cursor, Claude Code 같은 코딩 에이전트를 쓴다면 아래처럼 시키면 됩니다.

```text
이 저장소를 로컬에서 실행 가능한 상태로 세팅해줘.
README.md와 docs/setup/agent-assisted-setup.md를 기준으로 진행하고,
런타임 확인, pnpm install, pnpm docs:html, pnpm verify까지 실행해줘.
실제 브로커 비밀값이나 계좌번호는 만들거나 저장하지 말고,
막히는 부분만 나한테 물어봐.
```

## 저장소 구조

```text
apps/
  api/       로컬 API 서버
  desktop/   Electron + React 데스크톱 앱

packages/
  core/      Commander 실행 흐름, 권한 정책, 주문 차단, 결과물 저장
  db/        SQLite 구조와 저장 코드
  shared/    API와 UI가 함께 쓰는 타입

docs/
  architecture/  설계 문서와 코드 위치 지도
  stages/        단계별 완료 기준
  waterfall/     단계별 개발 계획
  handoffs/      긴 다음 작업 명세

vendor/
  tossinvest/    Toss Invest OpenAPI 로컬 사본
```

`external/`은 Git에 올리지 않습니다. 큰 참고 구현이나 선택형 보조 도구는 그 안에 두고, 이 저장소에는 필요한 연결 지점과 문서만 남기는 방식입니다.

## 권한 설계

GaemiGuard는 투자 기능을 한꺼번에 열지 않고 권한을 나눠서 다룹니다.

- 읽기: 계좌, 보유 종목, 가격, 환율, 일정, 경고를 가져옵니다.
- 정리: 투자 원칙, 리서치, 시나리오, 판단 이유를 로컬에 저장합니다.
- 검토: 주문 전후로 비중, 현금, 원칙 위반, 근거 부족을 확인합니다.
- 수동 실행: 사용자가 승인한 주문만 공식 브로커 API로 실행합니다.
- 자동 실행: 사용자가 정한 원칙, 한도, 쿨다운, 중단 스위치 안에서만 실행합니다.

로컬 파일이나 에이전트 실행 권한이 높아도, 그것이 곧바로 주문 권한이 되지는 않습니다. 주문 권한은 별도입니다.

## 보안과 투자 경계

GaemiGuard는 투자 판단을 돕는 도구입니다. 특정 수익률, 미래 가격, 매매 성과를 약속하지 않습니다.

실거래와 자동매매는 아래 조건을 통과한 흐름에서만 실행됩니다.

- 사용자의 명시 승인
- 중복 주문 방지 키
- 감사 로그
- 즉시 중단 스위치
- 계좌/종목/금액별 제한
- 실패했을 때의 명확한 중단 상태
- 비밀값 저장과 마스킹 정책

API 키, OAuth 비밀값, 계좌 식별자, 토큰, 원본 로그 같은 민감 정보는 이슈나 PR에 올리지 마세요. 보안 문제는 `SECURITY.md`를 따라 비공개로 제보해 주세요.

## 기여 방향

좋은 기여:

- 권한 정책과 감사 로그를 더 명확하게 만드는 변경
- 공식 브로커 API 기반 연결
- 투자 판단 근거를 저장하고 다시 확인하기 쉬운 결과물
- 화면에서 중요한 정보를 더 잘 보이게 만드는 변경
- 테스트 가능한 작은 런타임 개선

피해야 할 기여:

- 비공식 Toss 웹 내부 API 의존
- 승인 없는 실주문 실행
- 수익 보장처럼 보이는 문구
- 계좌번호, 토큰, 원본 로그가 들어간 테스트 자료
- 앱 전체를 무거운 Docker 실행에 묶는 구조
